<Aside type="caution">
  This project requires **Bun 1.1+** and a running **PostgreSQL** instance with the **pgvector** extension.
</Aside>

## Prerequisites

Before you begin, ensure you have the following installed:
- [Bun](https://bun.sh/)
- [PostgreSQL](https://www.postgresql.org/) (with `pgvector`)
- [Inngest Dev Server](https://www.inngest.com/docs/local-development)

## Step-by-Step Guide

<Steps>

1.  **Clone the Repository**
    ```bash
    git clone https://github.com/my-org/my-repo.git
    cd my-repo/ai-code-review
    ```

2.  **Install Dependencies**
    ```bash
    bun install
    ```

3.  **Configure Environment Variables**
    Create a `.env` file in the root of the `ai-code-review` directory.

    <Tabs>
      <TabItem label="Required">
        ```properties
        DATABASE_URL="postgresql://user:password@localhost:5432/dbname"
        GOOGLE_GENERATIVE_AI_API_KEY="your-gemini-key"
        GITHUB_CLIENT_ID="your-github-id"
        GITHUB_CLIENT_SECRET="your-github-secret"
        BETTER_AUTH_SECRET="your-auth-secret"
        ```
      </TabItem>
      <TabItem label="Optional">
        ```properties
        GROQ_API_KEY="your-groq-key"
        HF_TOKEN="your-huggingface-token"
        ```
      </TabItem>
    </Tabs>

4.  **Database Migration**
    Apply the Prisma schema to your local database.
    ```bash
    bun x prisma db push
    ```

5.  **Start Inngest Dev Server**
    In a separate terminal, start the Inngest local server.
    ```bash
    npx inngest-cli@latest dev
    ```

6.  **Run Development Server**
    ```bash
    bun dev
    ```

</Steps>

## Connecting GitHub Webhooks

To test PR reviews locally, you must use a tool like `ngrok` to expose your local server to GitHub.

1.  Start ngrok: `ngrok http 3000`
2.  Add the ngrok URL to your GitHub App's Webhook settings: `https://<your-id>.ngrok.app/api/webhooks/github`
3.  Ensure the `pull_request` event is selected.

<Aside type="tip">
  Use the **Inngest Cloud** for production deployments to ensure high availability and event durability.
</Aside>
